# Project Rules

## Git Push Permission

**IMPORTANT**: Before executing any `git push` command (to any remote or branch), you MUST explicitly ask for and receive the user's permission. Do NOT push automatically.

- Always explain what will be pushed (branch name, commit count, target remote/URL) before asking.
- Wait for the user to explicitly say yes, go ahead, push it, or similar approval.
- If the user says no or does not respond, abort the push.
- This rule applies to force pushes, normal pushes, and any variant of `git push`.
